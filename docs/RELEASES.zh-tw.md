# 發布政策

英文版為規範性版本：[RELEASES.md](RELEASES.md)。

Quick Tunnel Review Share 尚未發布帶有 Git 標籤的版本。第一次發布前，只有
`main` 是受支援的開發線，且尚未承諾穩定 API。

## 版本規則

正式 Git 標籤採用 Semantic Versioning。公開介面包含 CLI 參數、機器可讀事件
欄位、安裝路徑、支援平台，以及文件中承諾的安全邊界。

預定的第一個版本是 `v0.1.0`，但本文件與變更紀錄中的「尚未發布」項目都
不會自行建立發布版本。只有在完全相同的提交版本通過 CI 與發布檢查清單，
並取得明確發布決定後，才能建立 Git 標籤與 GitHub Release。

## 發布檢查清單

1. 在 `CHANGELOG.md` 及其繁體中文對照文件中加入版本與日期。
2. 對完全相同的候選版本執行完整 Windows 與原生 macOS 驗證。
3. 確認該提交版本的 GitHub Actions 檢查結果。
4. 只針對預計發布的來源檔檢查機密、私人網址、本機路徑、研究報告與規劃資料。
5. 確認相容性，或記錄每一項刻意的破壞性變更。
6. 建立或推送 Git 標籤、發布 GitHub Release 前，取得明確授權。
7. 確認公開 Git 標籤指向已核准的提交版本。

在第一次發布的範圍與發布授權確立前，暫不加入發布自動化。這可避免
一般 Git 標籤推送在未察覺的情況下直接形成公開發布版本。
