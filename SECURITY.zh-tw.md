# 安全政策

[英文權威版本](SECURITY.md)

## 支援版本

在正式發布帶有 Git 標籤的版本前，只支援 `main` 最新提交。舊提交與本機修改版
可能不包含目前的安全修正。

## 通報漏洞

如果 `pingqLIN/Quick-Tunnel` 已啟用 GitHub 私人漏洞通報功能（private vulnerability reporting），
請使用該私人通報管道；若無法使用，請在公開細節前先私下聯絡儲存庫
擁有者。不要在公開 GitHub Issue 貼出機密、仍有效的通道 URL、私人原始碼、
憑證，或會暴露他人暫存快照的利用方式。

通報內容應包含受影響平台與修訂版本、最小且已遮蔽敏感資訊的重現方式、
預期安全界線與實際行為。不得附上真實憑證或私人專案資料。

## 安全範圍

Quick Tunnel Review Share 是未經身分驗證、短時間使用的過濾快照傳輸工具，
不是私人檔案分享服務、完整 DLP（資料外洩防護）系統，也不是具身分驗證的
協作平台。保證、限制與高敏感資料的替代方案，請參閱
[威脅模型](docs/THREAT_MODEL.zh-tw.md)。
